6. WEBHOOKS: SUBSCRIPTION EVENTS
==================================================

Update the webhook system to handle subscription events, not only payment capture events.

Create/fix:
POST /api/paypal/webhook

Requirements:
- public route
- verify webhook signature securely
- use PAYPAL_WEBHOOK_ID
- add structured logging
- make processing idempotent
- invalid signature returns 400
- valid handled events return 200 quickly

Handle relevant subscription lifecycle events, including at minimum:
- BILLING.SUBSCRIPTION.CREATED
- BILLING.SUBSCRIPTION.ACTIVATED
- BILLING.SUBSCRIPTION.CANCELLED
- BILLING.SUBSCRIPTION.SUSPENDED
- BILLING.SUBSCRIPTION.EXPIRED
- PAYMENT.SALE.COMPLETED
- BILLING.SUBSCRIPTION.PAYMENT.FAILED

Behavior:
- On BILLING.SUBSCRIPTION.ACTIVATED:
  - set user.isPremium = true
  - set premiumActivatedAt = now
  - set premiumSource = "paypal"
  - save paypalSubscriptionId
  - save subscriptionStatus = active

- On PAYMENT.SALE.COMPLETED:
  - update lastPaymentAt
  - keep premium active if subscription is valid

- On BILLING.SUBSCRIPTION.CANCELLED:
  - set subscriptionStatus = cancelled
  - set user.isPremium = false

- On BILLING.SUBSCRIPTION.SUSPENDED:
  - set subscriptionStatus = suspended
  - set user.isPremium = false

- On BILLING.SUBSCRIPTION.EXPIRED:
  - set subscriptionStatus = expired
  - set user.isPremium = false

- On BILLING.SUBSCRIPTION.PAYMENT.FAILED:
  - update subscription status appropriately
  - if the subscription should no longer grant premium, disable premium

Make sure duplicate webhook deliveries do not cause duplicated updates.
